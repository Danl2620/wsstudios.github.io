Deciding on the future of TDP1's Version Control

Dan Liebgold
6/6/2020
[Github & Perforce]: This is the status quo. No real changes to our current workflow and licenses other than the migration of Perforce into our VPC.
+
<Branching>: Pervasive branching model supporting tasks and DevOps.
+
<Mature support>: Well established community of users and tools.
+
This is what we are used to and currently use. It entails the least effort towards new systems and workflows.
+
Git supports our submodule layout.
+
Integrates with other git repositories will (e.g. pragma)
+
Perforce fits the workflow for truly binary un-branched assets.
-
<Two systems>: This choice uses two version control systems.
-
Two subscriptions, two skillsets, two approaches.
-
The breakdown of what asset goes in which system isn't always clear and is the subject of ongoing debate.
-
If we put the UE4 project's `Content` directory into Perforce to help the artist and designer workflow, we'll saddle programmers with extra sync-up work as git branches get merged. Iâˆƒ we don't really realize the benefits of Perforce's binary file and lock handling.
-
Git is not supported by UE4.
-
<Git LFS>:
-
Doesn't support locking of assets
-
Has arcane failure modes (such as when there is a merge conflict)
-
And is very confusing for everyone, especially non-programmer.
[PlasticSCM]: A newer niche source control system that purports to combine the best of Git (distributed branches based version control) and Perforce/SVN (centralized trunk based version control) with good support for binary assets.
+
<Unified system>: single subscription and licensing approach.
+
<Branching>: Pervasive branching model supporting tasks and DevOps.
+
Provides task branches for code and a push-to-main model for content.
+
Good binary support.
+
Unified and high quality GUI tools (including artists-friendly client app).
+
Highly opinionated branching model (also a con).
+
Fantasic documentation about their product as well as version control and DevOps in general, see https:
+
Native application contains all features (i.e. no webapps).
-
New and unusual system, not used by many. We we likely hit corner cases that aren't well explored by the commmunity.
-
UE4 support is somewhat provisional
-
Highly opinionated branching model (also a pro).
[Perforce]: Switch to Perforce for everything, code included.
+
<Unified system>: single subscription and licensing approach.
+
<Mature support>: Well established community of users and tools.
+
Support push-to-main workflows suitable for content.
+
Good binary support.
+
"Streams" features adds branching support that may work for DevOps if not for tasks.
+
It's the industry standard.
+
Built-in support in UE4.
-
Poor (or at least unusual) support for feature or task branches.
-
Unclear approach to code review.
-
Poor tooling: p4v and command line only.
-
Centralized control -- difficult to follow distributed workflow (i.e. when working from home or disconnected).
